# Handoff: Портфолио-визитка «Анастасия» (генеративный художник)

## Overview
Одностраничный сайт-визитка / портфолио для художницы, работающей с генеративными
изображениями. Строгий, минималистичный, светлый «галерейный» стиль с большим
количеством воздуха. Двуязычный (RU/EN) с живым переключателем языка.

## About the Design Files
Файлы в этом бандле — **дизайн-референс, сделанный в HTML** (прототип, показывающий
задуманный вид и поведение), а не продакшн-код для прямого копирования. Задача —
**воссоздать этот дизайн в целевой кодовой базе** её собственными паттернами и
библиотеками. Если окружения ещё нет — выбрать подходящий фреймворк (для статики на
github.io отлично подойдёт чистый HTML/CSS/JS или Astro/Vite) и реализовать дизайн там.

Исходник построен как «Design Component» (`.dc.html`) — это формат прототипа. В нём
шаблон + класс логики. Переносить нужно не формат, а **разметку, стили и поведение**.

## Fidelity
**High-fidelity (hifi).** Финальные цвета, типографика, отступы, анимации и интерактив.
Воссоздавать пиксель-в-пиксель, подставив реальные изображения работ.

## Interactions & Behavior
- **Переключатель языка (RU/EN):** state `lang`; весь текст берётся из словаря `L[lang]`. Клик по RU/EN меняет `lang` и перерисовывает все тексты. Неактивный язык — `opacity:.35`.
- **Якорная навигация:** `html{scroll-behavior:smooth}`, ссылки ведут на `#work/#about/#services/#contact`.
- **Hover-состояния:** nav-подчёркивание (width 0→100%, .35s), зум изображений (scale 1.045, 1.1s), выезд подписи плитки, сдвиг строк услуг, подчёркивание контакт-ссылок.
- **Появление hero:** keyframe `rise` (opacity 0→1, translateY 20px→0, 1s `cubic-bezier(.2,.7,.2,1)`), задержки .05/.14/.24/.34s.
- **Адаптив:** брейкпоинт 820px схлопывает сетки в одну колонку и прячет главную навигацию.

## State Management
- `lang: 'ru' | 'en'` — единственное состояние. Всё остальное статично.
- Данные не фетчатся; тексты захардкожены в словаре RU/EN.

## Files
- `Анастасия — портфолио.dc.html` — полный прототип (разметка + стили + логика RU/EN).
- `index.html` — главная. Сетка работ строится сама из `projects/*.js` по массиву `WORKS`;
  плитка ведёт на `project.html?id=<id>`, на hover показывает описание + «Подробнее».
- `content.js` — тексты главной (RU/EN), подставляются по `data-i18n`.
- `project.html` — универсальная внутренняя страница проекта. Читает `?id=` из URL,
  подгружает `projects/<id>.js`, рендерит описание + карусель «Детали».
- `projects/*.js` — мета-файлы проектов (RU/EN), по одному на проект (`w1…w7`, `elevator`).
- `models.html` — длинная страница «Модели/Музы»: секции с чередованием фото ↔ текст,
  «летающие» имена в hero (клик → плавный скролл к модели).
- `models.js` — данные моделей (RU/EN), массив `window.MODELS`.
- `assets/Projects/<Name>/1.png … N.png` — кадры проектов, папка на проект
  (`Solar`, `Marrakech`, `Rose`, `Pearls`, `Studio`, `Wool`, `Ivory`, `ProjectElevator`).
- `assets/Models/<Name>/1.jpg` — фото моделей (`Claudia`, `Mila`, `Naomi`).

## Страницы проектов и мета-файлы
Мета-файл `projects/<id>.js` — **единственное место правды** по проекту. Из него берут
данные И плитка на главной (обложка, название, короткое описание, кнопка «Подробнее»
на hover), И внутренняя страница. Структура:

```js
// регистрируется в общий реестр — главная перебирает его для сетки
(window.PROJECTS = window.PROJECTS || {})["w5"] = window.PROJECT = {
  id: "w5",                 // = имя файла и ?id= в ссылке
  fig: "FIG.01",            // подпись-номер (угол плитки)
  orientation: "vertical",  // "vertical" | "horizontal" — высота карусели «Детали»
  image: "assets/Solar/1.png",   // обложка: плитка на главной + шапка страницы
  gallery: [                     // картинки карусели «Детали» (общие для RU/EN)
    "assets/Solar/1.png", "assets/Solar/2.png", "assets/Solar/3.png"
  ],
  ru: {
    title, year, meta, tags: [...],
    lead,                   // короткая подводка — она же текст на hover плитки
    captions: [...],        // заголовок кадра в карусели «Детали» (по порядку gallery[])
    notes: [...],           // текст справа от карусели — свой на каждый кадр (по gallery[])
    body: [...]             // абзацы описания
  },
  en: { /* то же на английском */ }
};
```

**Первая строка обязательна** (`(window.PROJECTS…)["<id>"] = window.PROJECT = {`) —
именно она добавляет проект в реестр, из которого строится главная галерея.

**Ассеты проекта (структура):** папка с именем проекта и нумерованными кадрами —
`assets/<Name>/1.jpg … N.jpg`. Примеры: `assets/ProjectElevator/1.jpg…5.jpg`
(проект `elevator`), `assets/Solar/1.png` (проект `w5`). Каждый проект — своя папка;
у `w1…w7` пока по одному кадру `1.png` (карусель тянет соседние проекты как демо —
замени на реальные `2.png, 3.png…` в своей папке).

**Добавить проект (плитку HTML писать НЕ надо — сетка строится сама):**
1. Сложить кадры в `assets/<Name>/1.jpg … N.jpg`.
2. Создать `projects/<id>.js` по образцу (первая строка с реестром + `image`, `gallery[]`,
   `captions[]` в каждом языке — по числу кадров, `lead` = короткое описание для hover).
3. Дописать `"<id>"` в массив `WORKS` в `index.html` (порядок плиток на главной).
4. Дописать `"<id>"` в массив `ORDER` в `project.html` (порядок стрелки «следующая»).

`id` — латиница/цифры/`-`/`_` (совпадает с именем файла в `projects/`).

**Плитки главной:** одинаковые вертикальные (`aspect-ratio 3/4`), сетка 3→2→1 колонки
по ширине экрана. На hover: затемнение, короткое описание (`lead`, RU/EN) и кнопка
«Подробнее / View» плавно проявляются. Название и год видны всегда.

**Карусель «Детали» (двухколоночная):** слева — вертикальная сцена кадра, справа —
текст. У каждого кадра свой текст (`captions[i]` = заголовок, `notes[i]` = абзац);
при смене слайда текст справа **плавно кроссфейдится** синхронно с картинкой. Большая
цифра справа = номер кадра. Управление (стрелки ←/→, точки) — под текстом; авто-прокрутка
4.6s с паузой на hover, свайп на тач, клавиши ←/→. Кадр слева показывается **целиком**
(`contain`, без обрезки), края заполняет размытый дубль того же кадра (тикток-стайл),
поэтому координаты кропа не нужны — любое фото влезает как есть. Поле `orientation`
в мете сейчас не влияет на «Детали» (сцена всегда вертикальная), оставлено на будущее.

## Страница «Модели» (models.html)
Длинная страница про постоянные образы-музы. Строится из `models.js` (`window.MODELS` —
массив в порядке отображения). Секции чередуют фото и текст (чёт/нечёт), появляются
скроллом (IntersectionObserver). В hero — «летающие» курсивные имена (лёгкая анимация
`floaty`), клик по имени плавно скроллит к якорю `#<id>` модели.

Структура записи модели:
```js
{
  id: "claudia",                      // = якорь #claudia
  name: "Claudia",
  image: "assets/Models/Claudia/1.jpg",
  ru: { role, meta, tags: [...], body: [...] },
  en: { /* то же */ }
}
```
**Добавить модель:** положить фото в `assets/Models/<Name>/1.jpg`, дописать объект в
массив `models.js`. Секция, «летающее» имя и якорь появятся сами. Позиции имён в hero
берутся из массива `POS` в `models.html` (циклично).

## Локальный просмотр
`?id=`/`&lang=` читаются из URL — открывать через локальный сервер, не `file://`
(иначе `project.html` не подгрузит мета-файл). Напр.: `python -m http.server` →
открыть `http://localhost:8000`. Деплой на GitHub Pages работает как есть (статика).
